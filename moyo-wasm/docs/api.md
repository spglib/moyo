# moyo-wasm API reference

JavaScript/WebAssembly API of [moyo](https://github.com/spglib/moyo), published as [@spglib/moyo-wasm](https://www.npmjs.com/package/@spglib/moyo-wasm). The package ships TypeScript definitions generated by wasm-bindgen (via tsify) directly from the Rust types. The module must be initialized before any function is called: import the default export `init` and call it with the URL (or bytes) of the `.wasm` file.

```ts
import init, { analyze_cell } from '@spglib/moyo-wasm'
import wasm_url from '@spglib/moyo-wasm/moyo_wasm_bg.wasm?url'

await init(wasm_url)
// now any exported function may be called
```

The package is built with `wasm-pack build --target web --release --scope spglib`. The generated entry module is `moyo_wasm.js` with the binary `moyo_wasm_bg.wasm`.

## Conventions

- Lattice matrices are passed and returned as a flat length-9 array `[f64; 9]` in row-major order: `[m00, m01, m02, m10, m11, m12, m20, m21, m22]`. They are wrapped in a `Lattice` object with a single `basis` field.
- Atomic positions are fractional coordinates, given as an array of length-3 arrays (`[number, number, number][]`).
- Atomic numbers are integers (`number[]`).
- Rotation matrices in operations are length-9 row-major arrays; translations are length-3 arrays.
- `analyze_cell` takes its cell as a JSON **string** (you build a `MoyoCell` object and `JSON.stringify` it). All other functions take and return plain JS objects / numbers directly.
- The `setting` argument of `analyze_cell` is a plain string: `"Standard"` or `"Spglib"` (case-insensitive match on `"spglib"`; any other value, including `"Unknown"`, falls back to `Standard`).
- The `setting` argument of `operations_from_number` and `operations_from_layer_number` is a tagged-union object: `{ type: "Standard" }`, `{ type: "Spglib" }`, or `{ type: "HallNumber", value: <number> }`.
- Single-variant Rust enums used as output fields (centering) are serialized as bare strings, for example `"P"`, `"C"`, `"F"`. The `MoyoDataset.angle_tolerance` field is a tagged union: `{ type: "Default" }` or `{ type: "Radian", value: <number> }`.
- Error behavior: functions that return `Result` on the Rust side **throw** a JS error on invalid input. `analyze_cell` throws with descriptive messages for malformed JSON, missing fields, or shapes that do not match the expected cell structure, and for internal moyo errors. The database-lookup functions throw on out-of-range or unknown index arguments. Functions that return arrays directly (`hall_symbol_entries_from_number`, `wyckoff_positions`, `layer_hall_symbol_entries_from_number`, `layer_wyckoff_positions`) do not throw; they return an empty array for unknown inputs.
- `symprec` is a distance tolerance in the same length units as the lattice basis. `angle_tolerance` (where reported) is in radians.

## Symmetry dataset (space group)

### analyze_cell

```ts
function analyze_cell(cell_json: string, symprec: number, setting: string): MoyoDataset
```

Analyze the crystal symmetry of a structure and return the full symmetry dataset.

Parameters:

- `cell_json`: a JSON string of a `MoyoCell` object (`JSON.stringify(cell)`). Must contain `lattice` (object with `basis`), `positions`, and `numbers`.
- `symprec`: distance tolerance for symmetry search.
- `setting`: `"Standard"` or `"Spglib"` (case-insensitive on `"spglib"`; other values fall back to `Standard`).

Throws if the JSON is invalid, a required field is missing, the shapes do not match a cell, or moyo fails to analyze the structure.

Returns a `MoyoDataset` object (see below).

#### MoyoCell (input/output type)

- `lattice: Lattice` - the lattice basis wrapper.
- `positions: [number, number, number][]` - fractional coordinates of each site.
- `numbers: number[]` - atomic number of each site.

#### Lattice (input/output type)

- `basis: number[]` - length-9 row-major lattice matrix `[m00, m01, m02, m10, m11, m12, m20, m21, m22]`.

#### MoyoOperation (output type)

- `rotation: number[]` - length-9 row-major rotation matrix.
- `translation: [number, number, number]` - translation vector.

#### AngleTolerance (output type)

Tagged union:

- `{ type: "Radian", value: number }` - tolerance in radians.
- `{ type: "Default" }` - the default tolerance was used.

#### MoyoDataset (output type)

- `number: number` - space-group ITA number (1 - 230).
- `hall_number: number` - Hall number of the matched setting (1 - 530).
- `hm_symbol: string` - Hermann-Mauguin symbol, for example `"P m -3 m"`.
- `operations: MoyoOperation[]` - symmetry operations of the input cell.
- `orbits: number[]` - index of the representative site for each input site (orbit / symmetry-equivalence mapping).
- `wyckoffs: string[]` - Wyckoff letter for each input site.
- `site_symmetry_symbols: string[]` - site-symmetry symbol for each input site.
- `std_cell: MoyoCell` - standardized conventional cell.
- `std_linear: number[]` - length-9 linear part of the transformation to the standardized cell.
- `std_origin_shift: [number, number, number]` - origin shift to the standardized cell.
- `std_rotation_matrix: number[]` - length-9 rigid rotation aligning the input to the standardized cell.
- `pearson_symbol: string` - Pearson symbol of the structure.
- `prim_std_cell: MoyoCell` - standardized primitive cell.
- `prim_std_linear: number[]` - length-9 linear part of the transformation to the primitive standardized cell.
- `prim_std_origin_shift: [number, number, number]` - origin shift to the primitive standardized cell.
- `mapping_std_prim: number[]` - mapping from sites of the input cell to sites of `prim_std_cell`.
- `symprec: number` - the symmetry tolerance that was used.
- `angle_tolerance: AngleTolerance` - the angle tolerance that was used.

### operations_from_number

```ts
function operations_from_number(number: number, setting: MoyoSetting, primitive: boolean): MoyoOperation[]
```

Return the symmetry operations for a space group identified by its ITA `number`.

Parameters:

- `number`: space-group ITA number (1 - 230).
- `setting`: a `MoyoSetting` tagged union (see below).
- `primitive`: if `true`, return operations of the primitive cell; if `false`, of the conventional cell.

Throws on an invalid number. Returns an array of `MoyoOperation`.

#### MoyoSetting (input type)

Tagged union:

- `{ type: "Spglib" }`
- `{ type: "Standard" }`
- `{ type: "HallNumber", value: number }`

### layer_hall_symbol_entry

```ts
function layer_hall_symbol_entry(hall_number: number): MoyoLayerHallSymbolEntry
```

Return the layer Hall-symbol entry for the given `hall_number` (1 - 116). Throws on an unknown Hall number.

### layer_hall_symbol_entries_from_number

```ts
function layer_hall_symbol_entries_from_number(number: number): MoyoLayerHallSymbolEntry[]
```

Return all layer Hall-symbol entries for the given layer-group `number` (1 - 80), ordered by Hall number; each entry is one setting of the layer group. Returns an empty array for an unknown number (does not throw).

### layer_wyckoff_positions

```ts
function layer_wyckoff_positions(hall_number: number): MoyoLayerWyckoffPosition[]
```

Return all Wyckoff positions for the given layer Hall number (1 - 116), ordered general-position-first then descending multiplicity. Returns an empty array for an unknown Hall number (does not throw).

### layer_arithmetic_crystal_class

```ts
function layer_arithmetic_crystal_class(arithmetic_number: number): MoyoLayerArithmeticCrystalClass
```

Return the layer arithmetic-crystal-class entry for the given `arithmetic_number` (1 - 43). Throws on an unknown arithmetic number.

### layer_group_type

```ts
function layer_group_type(number: number): MoyoLayerGroupType
```

Return the layer-group-type description for the given `number` (1 - 80). Throws on an unknown number.

#### MoyoLayerCentering (output type)

Single-variant enum serialized as a bare string. One of: `"P"`, `"C"`.

#### MoyoLayerHallSymbolEntry (output type)

- `hall_number: number` - layer Hall number (1 - 116).
- `number: number` - layer-group number (1 - 80).
- `arithmetic_number: number` - layer arithmetic-crystal-class number (1 - 43).
- `setting: string` - setting label (may be empty).
- `hall_symbol: string` - layer Hall symbol.
- `hm_short: string` - short Hermann-Mauguin symbol.
- `hm_full: string` - full Hermann-Mauguin symbol.
- `centering: MoyoLayerCentering` - centering (`"P"` or `"C"`).

#### MoyoLayerWyckoffPosition (output type)

- `multiplicity: number` - multiplicity in the conventional layer cell.
- `letter: string` - Wyckoff letter.
- `site_symmetry: string` - site-symmetry symbol.
- `coordinates: string` - representative coordinate triplet, for example `"x,y,z"`.

#### MoyoLayerArithmeticCrystalClass (output type)

- `arithmetic_number: number` - layer arithmetic-crystal-class number (1 - 43).
- `symbol: string` - arithmetic-crystal-class symbol, for example `"c211"`.
- `geometric_crystal_class: string` - geometric crystal class.
- `bravais_class: string` - layer Bravais class.
- `lattice_system: string` - layer lattice system, for example `"Oblique"`, `"Rectangular"`, `"Hexagonal"`.

#### MoyoLayerGroupType (output type)

- `number: number` - layer-group number (1 - 80).
- `hall_number: number` - layer Hall number for the default setting.
- `hm_short: string` - short Hermann-Mauguin symbol.
- `hm_full: string` - full Hermann-Mauguin symbol.
- `arithmetic_number: number` - layer arithmetic-crystal-class number.
- `arithmetic_symbol: string` - arithmetic-crystal-class symbol.
- `geometric_crystal_class: string` - geometric crystal class.
- `crystal_system: string` - crystal system.
- `bravais_class: string` - layer Bravais class.
- `lattice_system: string` - layer lattice system.

## Magnetic space group

### magnetic_operations_from_uni_number

```ts
function magnetic_operations_from_uni_number(uni_number: number, primitive: boolean): MoyoMagneticOperation[]
```

Return the magnetic symmetry operations for the given `uni_number` (1 - 1651).

Parameters:

- `uni_number`: UNI (BNS) number of the magnetic space group (1 - 1651).
- `primitive`: if `true`, return primitive-cell operations; if `false`, conventional-cell operations.

Throws on an invalid `uni_number`. Returns an array of `MoyoMagneticOperation`.

### magnetic_hall_symbol_entry

```ts
function magnetic_hall_symbol_entry(uni_number: number): MoyoMagneticHallSymbolEntry
```

Return the magnetic Hall-symbol entry for the given `uni_number` (1 - 1651). Throws on an unknown `uni_number`.

### magnetic_space_group_type

```ts
function magnetic_space_group_type(uni_number: number): MoyoMagneticSpaceGroupType
```

Return the magnetic-space-group-type description for the given `uni_number` (1 - 1651). Throws on an unknown `uni_number`.

#### MoyoMagneticOperation (output type)

- `rotation: number[]` - length-9 row-major rotation matrix.
- `translation: [number, number, number]` - translation vector.
- `time_reversal: boolean` - whether the operation includes time reversal.

#### MoyoMagneticHallSymbolEntry (output type)

- `uni_number: number` - UNI number (1 - 1651).
- `magnetic_hall_symbol: string` - magnetic Hall symbol, for example `"P 1"`, `"P 1 1'"`.

#### MoyoMagneticSpaceGroupType (output type)

- `uni_number: number` - UNI number (1 - 1651).
- `litvin_number: number` - Litvin number.
- `bns_number: string` - BNS number, for example `"1.1"`.
- `og_number: string` - OG number, for example `"1.1.1"`.
- `number: number` - ITA space-group number of the underlying group.
- `construct_type: number` - magnetic-group construction type, an integer `1`, `2`, `3`, or `4`.

```ts
function space_group_type(number: number): MoyoSpaceGroupType
```

Return the space-group-type description for the given ITA `number` (1 - 230). Throws on an unknown number.

#### MoyoCentering (output type)

Single-variant enum serialized as a bare string. One of: `"P"`, `"A"`, `"B"`, `"C"`, `"I"`, `"R"`, `"F"`.

#### MoyoHallSymbolEntry (output type)

- `hm_short: string` - short Hermann-Mauguin symbol.
- `hm_full: string` - full Hermann-Mauguin symbol.
- `centering: MoyoCentering` - centering (`"P"`, `"A"`, `"B"`, `"C"`, `"I"`, `"R"`, `"F"`).

- `geometric_crystal_class: string` - geometric crystal class, for example `"3m"`.
- `bravais_class: string` - Bravais class, for example `"hP"`.

#### MoyoWyckoffPosition (output type)

- `multiplicity: number` - multiplicity in the conventional cell.
- `letter: string` - Wyckoff letter.
- `site_symmetry: string` - site-symmetry symbol.
- `coordinates: string` - representative coordinate triplet, for example `"x,y,z"`.

#### MoyoSpaceGroupType (output type)

- `number: number` - ITA space-group number (1 - 230).
- `hall_number: number` - Hall number of the default setting.
- `hm_short: string` - short Hermann-Mauguin symbol.
- `hm_full: string` - full Hermann-Mauguin symbol.
- `arithmetic_number: number` - arithmetic-crystal-class number.
- `arithmetic_symbol: string` - arithmetic-crystal-class symbol.
- `geometric_crystal_class: string` - geometric crystal class.
- `crystal_system: string` - crystal system, for example `"Cubic"`, `"Trigonal"`.
- `bravais_class: string` - Bravais class.
