# moyoc C API reference

C API of [moyo](https://github.com/spglib/moyo). The canonical header is the `moyoc.h` generated by cbindgen at build time; this page is a hand-maintained reference of it.

## Conventions

- 3x3 matrices and basis vectors are passed as row-major `double (*)[3]` arrays, where `basis[i]` is the `i`th row (the `i`th basis vector). Inside returned structs the same shapes appear as `double[3][3]`.
- Integer counts and index arrays use `int32_t`. Array lengths are given by a sibling count field (e.g. `num_atoms` or `num_operations`).
- Boolean arguments and fields use C `bool` (`<stdbool.h>`).
- Constructors (`*_new` functions) return `NULL` on failure (invalid arguments or a failed symmetry search). Always check for `NULL` before dereferencing.
- Every `Moyo*` object returned by a `*_new` function must be freed with its matching `*_free` function exactly once. Passing `NULL` to a `*_free` function is a no-op.
- Strings (`const char *`) inside returned structs are NUL-terminated and owned by the struct; they are freed by the struct's `*_free` call. Do not free them separately.
- Pointer fields inside a dataset (positions, numbers, operations, orbits, etc.) are owned by the dataset and freed by the dataset's `*_free` call. Do not call `moyo_operations_free` / `moyo_magnetic_operations_free` on operations embedded in a dataset.
- `moyo_version()` returns a statically allocated string that must not be freed.
- `angle_tolerance` arguments are in radians; pass a negative value to use the default. In returned structs, `angle_tolerance == -1` means the default angle tolerance was used.
- `mag_symprec` arguments: pass a negative value to reuse `symprec`.
- `epsilon` arguments (identification functions): pass a negative value to use the default tolerance (`1e-4`).

## Version

```c
const char *moyo_version(void);
```

Return the version string of moyoc (e.g. `"0.11.0"`).

- Returns: a statically allocated NUL-terminated string that must not be freed.

## Space-group dataset

### moyo_dataset_new

```c
MoyoDataset *moyo_dataset_new(const double (*basis)[3],
                              const double (*positions)[3],
                              const int32_t *numbers,
                              int32_t num_atoms,
                              double symprec,
                              double angle_tolerance,
                              MoyoSetting setting,
                              int32_t hall_number,
                              bool rotate_basis);
```

Analyze the symmetry of the given cell and create a dataset.

- `basis`: row-wise basis vectors, `basis[i]` is the `i`th basis vector (3 rows).
- `positions`: fractional coordinates of the `num_atoms` sites.
- `numbers`: atomic numbers of the `num_atoms` sites.
- `num_atoms`: number of atoms in the cell.
- `symprec`: tolerance for distance comparisons in symmetry search.
- `angle_tolerance`: tolerance of angle between basis vectors in radians. Pass a negative value to use the default angle tolerance.
- `setting`: preference for the setting of the space group (see `MoyoSetting`).
- `hall_number`: preference for Hall symbol, only used with `MOYO_SETTING_HALL_NUMBER`.
- `rotate_basis`: whether to apply the rigid rotation to the standardized basis.
- Returns: a new `MoyoDataset *`, or `NULL` if the symmetry search fails or the arguments are invalid. Free with `moyo_dataset_free`.

### moyo_dataset_free

```c
void moyo_dataset_free(MoyoDataset *dataset);
```

Free a dataset created by `moyo_dataset_new`. Passing `NULL` is a no-op.

- `dataset`: a pointer returned by `moyo_dataset_new` that has not been freed yet, or `NULL`.

### struct MoyoDataset

A dataset of the symmetry analysis, created by `moyo_dataset_new` and freed by `moyo_dataset_free`.

- `int32_t number`: space group number.
- `int32_t hall_number`: Hall symbol number.
- `const char *hm_symbol`: Hermann-Mauguin symbol in short notation (e.g. "Fd-3m" for space group 227).
- `int32_t num_atoms`: number of atoms in the input cell. Arrays `orbits`, `wyckoffs`, `site_symmetry_symbols`, and `mapping_std_prim` have this length.
- `MoyoOperations operations`: symmetry operations in the input cell.
- `const int32_t *orbits`: length `num_atoms`. The `i`th atom in the input cell is equivalent to the `orbits[i]`th atom in the input cell (spglib's `crystallographic_orbits`, not `equivalent_atoms`).
- `const char *wyckoffs`: Wyckoff letters for each site in the input cell as a single NUL-terminated string of length `num_atoms`.
- `const char *const *site_symmetry_symbols`: length `num_atoms`. Site symmetry symbol for each site; the orientation is w.r.t. the standardized cell.
- `MoyoCell std_cell`: standardized cell.
- `double std_linear[3][3]`: linear part of the transformation from the input cell to the standardized cell.
- `double std_origin_shift[3]`: origin shift of the transformation from the input cell to the standardized cell.
- `double std_rotation_matrix[3][3]`: rigid rotation.
- `const char *pearson_symbol`: Pearson symbol for the standardized cell.
- `MoyoCell prim_std_cell`: primitive standardized cell.
- `double prim_std_linear[3][3]`: linear part of the transformation from the input cell to the primitive standardized cell.
- `double prim_std_origin_shift[3]`: origin shift of the transformation from the input cell to the primitive standardized cell.
- `const int32_t *mapping_std_prim`: length `num_atoms`. The `i`th atom in the input cell is mapped to the `mapping_std_prim[i]`th atom in the primitive standardized cell.
- `double symprec`: actually used `symprec` in the iterative symmetry search.
- `double angle_tolerance`: actually used `angle_tolerance` in the iterative symmetry search; `-1` if the default angle tolerance was used.

### struct MoyoCell

Crystal structure. All pointer fields are owned by the containing `MoyoDataset` and are freed by `moyo_dataset_free`.

- `double basis[3][3]`: row-wise basis vectors.
- `const double (*positions)[3]`: length `num_atoms`. Fractional coordinates of the sites.
- `const int32_t *numbers`: length `num_atoms`. Atomic numbers of the sites.
- `int32_t num_atoms`: number of atoms in the cell.

### struct MoyoOperations

Symmetry operations. All pointer fields are owned by the containing `MoyoDataset` and are freed by `moyo_dataset_free`.

- `const int32_t (*rotations)[3][3]`: length `num_operations`. Rotation parts of the operations.
- `const double (*translations)[3]`: length `num_operations`. Translation parts of the operations.
- `int32_t num_operations`: number of symmetry operations.

## Layer-group dataset

### moyo_layer_dataset_new

```c
MoyoLayerDataset *moyo_layer_dataset_new(const double (*basis)[3],
                                         const double (*positions)[3],
                                         const int32_t *numbers,
                                         int32_t num_atoms,
                                         double symprec,
                                         double angle_tolerance,
                                         MoyoLayerSetting setting,
                                         int32_t hall_number,
                                         bool rotate_basis);
```

Analyze the layer-group symmetry of the given cell and create a dataset.

- `basis`: row-wise basis vectors, `basis[i]` is the `i`th basis vector. The third basis vector must be the aperiodic stacking direction, perpendicular to the first two, which must lie in the xy-plane.
- `positions`: fractional coordinates of the `num_atoms` sites.
- `numbers`: atomic numbers of the `num_atoms` sites.
- `num_atoms`: number of atoms in the cell.
- `symprec`: tolerance for distance comparisons in symmetry search.
- `angle_tolerance`: tolerance of angle between basis vectors in radians. Pass a negative value to use the default angle tolerance.
- `setting`: preference for the layer Hall setting (see `MoyoLayerSetting`).
- `hall_number`: preference for layer Hall symbol, only used with `MOYO_LAYER_SETTING_HALL_NUMBER`.
- `rotate_basis`: whether to apply the rigid rotation to the standardized basis.
- Returns: a new `MoyoLayerDataset *`, or `NULL` if the symmetry search fails or the arguments are invalid. Free with `moyo_layer_dataset_free`.

### moyo_layer_dataset_free

```c
void moyo_layer_dataset_free(MoyoLayerDataset *dataset);
```

Free a dataset created by `moyo_layer_dataset_new`. Passing `NULL` is a no-op.

- `dataset`: a pointer returned by `moyo_layer_dataset_new` that has not been freed yet, or `NULL`.

### struct MoyoLayerDataset

A dataset of the layer-group symmetry analysis, created by `moyo_layer_dataset_new` and freed by `moyo_layer_dataset_free`.

- `int32_t number`: layer group number (1 - 80).
- `int32_t hall_number`: layer Hall symbol number (1 - 116).
- `const char *hm_symbol`: Hermann-Mauguin symbol in short notation.
- `int32_t num_atoms`: number of atoms in the input cell. Arrays `orbits`, `wyckoffs`, `site_symmetry_symbols`, and `mapping_std_prim` have this length.
- `MoyoOperations operations`: layer-group operations in the input cell.
- `const int32_t *orbits`: length `num_atoms`. The `i`th atom in the input cell is equivalent to the `orbits[i]`th atom in the input cell.
- `const char *wyckoffs`: Wyckoff letters for each site in the input cell as a single NUL-terminated string of length `num_atoms`.
- `const char *const *site_symmetry_symbols`: length `num_atoms`. Site symmetry symbol for each site; orientation is w.r.t. the standardized cell.
- `MoyoCell std_cell`: conventional standardized layer cell.
- `double std_linear[3][3]`: linear part of the transformation from the input cell to the standardized cell.
- `double std_origin_shift[3]`: origin shift of the transformation from the input cell to the standardized cell.
- `double std_rotation_matrix[3][3]`: rigid rotation.
- `const char *pearson_symbol`: Pearson symbol for the standardized layer cell. The first two characters are the 2D Bravais type (`mp`, `op`, `oc`, `tp`, or `hp`).
- `MoyoCell prim_std_cell`: primitive standardized layer cell.
- `double prim_std_linear[3][3]`: linear part of the transformation from the input cell to the primitive standardized cell.
- `double prim_std_origin_shift[3]`: origin shift of the transformation from the input cell to the primitive standardized cell.
- `const int32_t *mapping_std_prim`: length `num_atoms`. The `i`th atom in the input cell is mapped to the `mapping_std_prim[i]`th atom in the primitive standardized cell.
- `double symprec`: actually used `symprec` in the iterative symmetry search.
- `double angle_tolerance`: actually used `angle_tolerance`; `-1` if the default angle tolerance was used.

## Magnetic space-group datasets

### moyo_collinear_magnetic_dataset_new

```c
MoyoCollinearMagneticDataset *moyo_collinear_magnetic_dataset_new(const double (*basis)[3],
                                                                  const double (*positions)[3],
                                                                  const int32_t *numbers,
                                                                  const double *magnetic_moments,
                                                                  int32_t num_atoms,
                                                                  double symprec,
                                                                  double angle_tolerance,
                                                                  double mag_symprec,
                                                                  bool is_axial,
                                                                  bool rotate_basis);
```

Analyze the magnetic symmetry of the given collinear magnetic cell and create a dataset.

- `basis`: row-wise basis vectors, `basis[i]` is the `i`th basis vector.
- `positions`: fractional coordinates of the `num_atoms` sites.
- `numbers`: atomic numbers of the `num_atoms` sites.
- `magnetic_moments`: collinear magnetic moments of the `num_atoms` sites (one `double` per site).
- `num_atoms`: number of atoms in the cell.
- `symprec`: tolerance for distance comparisons.
- `angle_tolerance`: tolerance of angle between basis vectors in radians. Pass a negative value to use the default angle tolerance.
- `mag_symprec`: tolerance for matching magnetic moments. Pass a negative value to reuse `symprec`.
- `is_axial`: whether magnetic moments transform as axial vectors.
- `rotate_basis`: whether to apply the rigid rotation to the standardized basis.
- Returns: a new `MoyoCollinearMagneticDataset *`, or `NULL` if the symmetry search fails or the arguments are invalid. Free with `moyo_collinear_magnetic_dataset_free`.

### moyo_collinear_magnetic_dataset_free

```c
void moyo_collinear_magnetic_dataset_free(MoyoCollinearMagneticDataset *dataset);
```

Free a dataset created by `moyo_collinear_magnetic_dataset_new`. Passing `NULL` is a no-op.

- `dataset`: a pointer returned by `moyo_collinear_magnetic_dataset_new` that has not been freed yet, or `NULL`.

- `angle_tolerance`: tolerance of angle between basis vectors in radians. Pass a negative value to use the default angle tolerance.

A dataset of the magnetic symmetry analysis of a collinear magnetic structure, created by `moyo_collinear_magnetic_dataset_new` and freed by `moyo_collinear_magnetic_dataset_free`.

- `MoyoCollinearMagneticCell std_mag_cell`: standardized magnetic cell.
- `double std_linear[3][3]`: linear part of the transformation from the input magnetic cell to the standardized one.
- `double std_origin_shift[3]`: origin shift of the transformation from the input magnetic cell to the standardized one.
- `double std_rotation_matrix[3][3]`: rigid rotation.
- `MoyoCollinearMagneticCell prim_std_mag_cell`: primitive standardized magnetic cell.
- `double prim_std_linear[3][3]`: linear part of the transformation from the input magnetic cell to the primitive standardized one.
- `double prim_std_origin_shift[3]`: origin shift of the transformation from the input magnetic cell to the primitive standardized one.
- `const int32_t *mapping_std_prim`: length `num_atoms`. The `i`th atom in the input magnetic cell is mapped to the `mapping_std_prim[i]`th atom in the primitive standardized magnetic cell.
- `double symprec`: actually used `symprec` in the iterative symmetry search.
- `double angle_tolerance`: actually used `angle_tolerance`; `-1` if the default angle tolerance was used.
- `double mag_symprec`: actually used `mag_symprec` in the iterative symmetry search.

### struct MoyoNonCollinearMagneticDataset

A dataset of the magnetic symmetry analysis of a non-collinear magnetic structure, created by `moyo_noncollinear_magnetic_dataset_new` and freed by `moyo_noncollinear_magnetic_dataset_free`. Field layout is identical to `MoyoCollinearMagneticDataset` except the magnetic cells are non-collinear.

- `int32_t uni_number`: serial number of UNI (and BNS) symbols.
- `int32_t num_atoms`: number of atoms in the input magnetic cell. Arrays `orbits` and `mapping_std_prim` have this length.
- `MoyoMagneticOperations magnetic_operations`: magnetic symmetry operations in the input cell.
- `const int32_t *orbits`: length `num_atoms`. The `i`th atom in the input magnetic cell is equivalent to the `orbits[i]`th atom in the input magnetic cell.
- `MoyoNonCollinearMagneticCell std_mag_cell`: standardized magnetic cell.
- `double std_linear[3][3]`: linear part of the transformation from the input magnetic cell to the standardized one.
- `double std_origin_shift[3]`: origin shift of the transformation from the input magnetic cell to the standardized one.
- `double std_rotation_matrix[3][3]`: rigid rotation.
- `MoyoNonCollinearMagneticCell prim_std_mag_cell`: primitive standardized magnetic cell.
- `double prim_std_linear[3][3]`: linear part of the transformation from the input magnetic cell to the primitive standardized one.
- `double prim_std_origin_shift[3]`: origin shift of the transformation from the input magnetic cell to the primitive standardized one.
- `const int32_t *mapping_std_prim`: length `num_atoms`. The `i`th atom in the input magnetic cell is mapped to the `mapping_std_prim[i]`th atom in the primitive standardized magnetic cell.
- `double symprec`: actually used `symprec` in the iterative symmetry search.
- `double angle_tolerance`: actually used `angle_tolerance`; `-1` if the default angle tolerance was used.
- `double mag_symprec`: actually used `mag_symprec` in the iterative symmetry search.

### struct MoyoCollinearMagneticCell

Collinear magnetic structure. All pointer fields are owned by the containing dataset and are freed by `moyo_collinear_magnetic_dataset_free`.

- `MoyoCell cell`: crystal structure.
- `const double *magnetic_moments`: length `cell.num_atoms`. Collinear magnetic moments of the sites.

### struct MoyoNonCollinearMagneticCell

Non-collinear magnetic structure. All pointer fields are owned by the containing dataset and are freed by `moyo_noncollinear_magnetic_dataset_free`.

- `MoyoCell cell`: crystal structure.
- `const double (*magnetic_moments)[3]`: length `cell.num_atoms`. Non-collinear magnetic moments of the sites in Cartesian coordinates.

### struct MoyoMagneticOperations

Magnetic symmetry operations. All pointer fields are owned by the containing dataset and are freed together with it.

- `const int32_t (*rotations)[3][3]`: length `num_operations`. Rotation parts of the magnetic operations.
- `const double (*translations)[3]`: length `num_operations`. Translation parts of the magnetic operations.
- `const bool *time_reversals`: length `num_operations`. Time-reversal parts of the magnetic operations.
- `int32_t num_operations`: number of magnetic symmetry operations.

## Symmetry operations from the database

### moyo_operations_from_number

```c
MoyoOperations *moyo_operations_from_number(int32_t number,
                                            MoyoSetting setting,
                                            int32_t hall_number,
                                            bool primitive);
```

Return symmetry operations for the given space-group ITA `number` (1 - 230).

- `number`: ITA space-group number (1 - 230).
- `setting`: preference for the setting (see `MoyoSetting`).
- `hall_number`: preference for Hall symbol, only used with `MOYO_SETTING_HALL_NUMBER` (`number` is ignored in that case).
- `primitive`: if true, return operations of the primitive cell; otherwise return operations of the conventional cell.
- Returns: a new `MoyoOperations *`, or `NULL` if the arguments are invalid. Free with `moyo_operations_free`.

### moyo_operations_from_layer_number

```c
MoyoOperations *moyo_operations_from_layer_number(int32_t number,
                                                  MoyoLayerSetting setting,
                                                  int32_t hall_number,
                                                  bool primitive);
```

Return symmetry operations for the given layer-group `number` (1 - 80).

- `number`: layer-group number (1 - 80).
- `setting`: preference for the layer Hall setting (see `MoyoLayerSetting`).
- `hall_number`: preference for layer Hall symbol, only used with `MOYO_LAYER_SETTING_HALL_NUMBER` (`number` is ignored in that case).
- `primitive`: if true, return operations of the primitive cell; otherwise return operations of the conventional cell.
- Returns: a new `MoyoOperations *`, or `NULL` if the arguments are invalid. Free with `moyo_operations_free`.

### moyo_operations_free

```c
void moyo_operations_free(MoyoOperations *operations);
```

Free operations returned by `moyo_operations_from_number` or `moyo_operations_from_layer_number`. Passing `NULL` is a no-op. Do not call this on operations embedded in a dataset; those are freed by the dataset's free function.

- `operations`: a pointer returned by `moyo_operations_from_number` or `moyo_operations_from_layer_number` that has not been freed yet, or `NULL`.

### moyo_magnetic_operations_from_uni_number

```c
MoyoMagneticOperations *moyo_magnetic_operations_from_uni_number(int32_t uni_number,
                                                                 bool primitive);
```

Return magnetic symmetry operations for the given UNI (and BNS) serial number `uni_number` (1 - 1651).

- `uni_number`: UNI (and BNS) serial number (1 - 1651).
- `primitive`: if true, return magnetic operations of the primitive cell; otherwise return operations of the conventional cell.
- Returns: a new `MoyoMagneticOperations *`, or `NULL` if the arguments are invalid. Free with `moyo_magnetic_operations_free`.

### moyo_magnetic_operations_free


## Enums

### enum MoyoSetting

Preference for the setting of the space group.

- `MOYO_SETTING_HALL_NUMBER`: Hall number specified by the `hall_number` argument.
- `MOYO_SETTING_SPGLIB`: the setting of the smallest Hall number.
- `MOYO_SETTING_STANDARD`: unique axis b, cell choice 1 for monoclinic, hexagonal axes for rhombohedral, and origin choice 2 for centrosymmetric space groups.

### enum MoyoLayerSetting

Preference for the Hall setting of the layer group.

- `MOYO_LAYER_SETTING_HALL_NUMBER`: layer Hall number specified by the `hall_number` argument (1 - 116).
- `MOYO_LAYER_SETTING_SPGLIB`: the setting of the smallest layer Hall number for each layer group (origin choice 1 for the centrosymmetric layer groups 52, 62, 64).
- `MOYO_LAYER_SETTING_STANDARD`: BCS standard choice (de la Flor et al., Acta Cryst. A77, 559-571 (2021)): origin choice 2 for the centrosymmetric layer groups 52, 62, 64.
